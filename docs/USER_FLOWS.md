# User Flows

## 项目知识问答（目标流程，当前 Mock）

当前只使用授权后按 `projectId` 过滤的 Mock 知识与引用，不接受真实文件，也没有解析、索引或 RAG。

```mermaid
flowchart LR
  A["进入项目"] --> B["打开项目知识"]
  B --> C["输入问题"]
  C --> D["Query Router"]
  D --> E["项目与权限过滤"]
  E --> F["版本过滤"]
  F --> G["检索"]
  G --> H["生成回答"]
  H --> I["展示来源引用"]
  I --> J["打开来源详情"]
```

验收重点：请求必须绑定 projectId；只使用当前有效知识；回答包含文件、章节、页码或片段；来源可以打开；无权限时不得泄露任何片段。

## 需求提取与审核（目标流程，当前 Mock）

当前只演示 Mock AI 草稿、审核交互和状态反馈；选择文件、解析和正式需求写入均未实现。

```mermaid
flowchart LR
  A["选择项目文件"] --> B["启动需求提取"]
  B --> C["文档解析"]
  C --> D["需求提取"]
  D --> E["重复与冲突识别"]
  E --> F["生成验收标准"]
  F --> G["形成 AI 草稿"]
  G --> H["人工修改"]
  H --> I["提交审核"]
  I --> J{"审核决定"}
  J -->|通过或修改后通过| K["目标：写入正式需求（当前未实现）"]
  J -->|驳回| L["返回工作流修订"]
```

目标验收重点：AI 输出始终是草稿；Failure 可重试；原始结果、修改差异、审核备注和 executionId 可追溯；未来正式写入只能由人工通过触发。v0.3 不包含正式需求数据层。
