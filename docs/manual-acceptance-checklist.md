# 项目经理日报 MVP 人工验收清单

## 前置门禁

- [ ] 使用 Local/Staging，绝不使用 Production。
- [ ] Migration 0016 和升级测试已通过；数据库已有可恢复备份。
- [ ] 两个 Feature Flag 只在验收环境显式开启。
- [ ] 使用虚构项目和虚构任务；没有真实客户资料或凭据进入证据。
- [ ] 扩展 manifest 没有 `<all_urls>`，未绑定未知企业微信 Origin。
- [ ] 用户负责手动登录；不记录 Cookie、Token 或二维码。

## ProjectAI

- [ ] 导航显示“工作日报”，Flag 关闭时路由/API 均拒绝。
- [ ] 随记可回车或按钮保存，失败时输入保留；可编辑、删除和手动修改时间。
- [ ] 无记录时 AI 按钮不可用且无 Provider execution。
- [ ] AI 只返回授权项目，缺工时/低置信字段进入审核。
- [ ] 计划或讨论中的事实没有被改成完成；总工时异常仅警告。
- [ ] 删除、拆分、同项目/分类/状态合并、保存草稿均可用。
- [ ] 未审核任务不能确认；并发旧版本返回冲突。
- [ ] 确认后可复制/下载 JSON；修改源随记使未同步确认失效。
- [ ] 其他用户、其他组织、失去项目权限和伪造资源 ID 均返回 404/受控拒绝。

## 扩展十轮验收

1. [ ] Mock 页面 Dry Run：字段全部匹配，单条保存与最终提交计数均为 0。
2. [ ] Mock 页面实际保存：逐条成功，最终提交计数为 0。
3. [ ] 真实企业微信页面 Dry Run：用户手动登录，只验证一条虚构任务。
4. [ ] 真实页面实际保存一条虚构测试任务；人工确认没有最终提交。
5. [ ] 真实页面实际保存两至三条虚构测试任务，逐条结果正确。
6. [ ] 登录失效：进入 waiting_for_login，重新登录后由用户主动继续。
7. [ ] 保存失败：当前批次暂停，不继续盲目创建后续项。
8. [ ] 重复同步：saved 项被跳过，changed replay 被拒绝。
9. [ ] 浏览器/Service Worker 重启：running 变 unknown，不自动重建。
10. [ ] 人工核对 unknown：确认已保存后永久跳过；确认未保存后转 failed，只有主动继续才重试；全程未触碰最终提交。

## 结果与清理

- [ ] ProjectAI 显示当前项、总数、成功/失败/未知、等待/暂停/取消与历史。
- [ ] 后端审计含关键事件但无任务全文、凭据或完整页面数据。
- [ ] 下载的同步结果和扩展错误日志已脱敏。
- [ ] Popup 清除记录要求二次确认。
- [ ] 删除虚构测试任务/日报测试数据；证据只保留批准的脱敏截图和摘要。
